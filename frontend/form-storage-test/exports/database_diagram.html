Database Structure:

This diagram shows the hierarchical structure of the question bank database:

  • Question Banks are the top-level entities containing one or more pages.
  • Each Page contains multiple cards and has metadata like exam language and type.
  • Cards serve as containers that can hold either question content or material content.
  • Content items are specialized by type:
    • Question types: single choice, multiple choice, fill-in-blank, matching, long text, audio response
    • Material types: text, multimedia, LLM session

Relationships: The database uses a hierarchical one-to-many relationship structure where each parent entity can have multiple child entities (represented by the arrows in the diagram).

Complex Data: Complex data structures (options, media references, settings) are stored as JSON in text columns like options_data, media_data, and settings_data.
